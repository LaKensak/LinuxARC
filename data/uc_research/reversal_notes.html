Arc Raiders (PioneerGame.exe) - Reversal Notes

Spent about a week going through Arc Raiders so far. Dumped the game and dug through everything I could find. Internal codename is "Pioneer" if you see that referenced anywhere. This covers the anti-cheat stack and SDK development info. I will not discuss specifics about internal/external development as I am still working on making my own internal source to release.

Binary is ~242MB. Dump hash is c4f13b07bfcbece09dee7357ee0199433f3b3ecefc90e2c3250a3c7a23cb9318 if you want to compare with what you download.

I did not use an Unreal Dumper, only due to the fact that someone had told me Arc uses a heavily modified version of Unreal and that it would be more realistic to map the SDK myself.

Part of this was also used for my research in my runtime analysis of EAC.

I do want to thank @Nknights23 for not gatekeeping knowledge for Arc and his DMA source, I did get some understanding of the internals from there. This post is mainly for those who want to do an internal for the game (or external if you really want) for whatever reason.

Anti-Cheat Architecture

This game has a ridiculous multi-layer AC stack. It has not only EAC, but there's behavioral ML, ONNX inference, custom Embark modules, and early-load anti-tamper.

Global State Flags:
  byte_7FF7B7963590  - Cerebro enabled flag (must be 1 to initialize)
  dword_7FF7B7963598 - Init counter (interlocked increment)
  qword_7FF7B793F9E8 - Stores first parameter (context pointer)
  dword_7FF7B793F9F0 - Stores API call result
  qword_7FF7B793F9F8 - Secondary API call result
  qword_7FF7B793F9D0 - Context for API call
  qword_7FF7B793F9E0 - Context for timeout operation (60s wait)
Crash Handler Integration:
Cerebro integrates with crash reporting at sub_7FF7AA099A50:
- Checks byte_7FF7B7963590 == 1 (Cerebro enabled)
- References qword_7FF7B793FA00 for crash context
- Calls sub_7FF7AA092CA0 for crash dumps
- String "Crash during static initialization" @ 0x7FF7AA099AEE
- Exit code 777006 used for Cerebro-related crashes

What it monitors:
  • Mouse movement vectors - acceleration curves, velocity patterns
  • Aim patterns - lock-on detection, snap detection, smoothing artifacts
  • Input timing - key press intervals, reaction time distributions
  • Movement prediction - humanness scoring via entropy analysis

For bypassing detection: your input patterns need human-like variance. No perfect tracking, no instant snaps, as these flag you immediately. Include natural micro-corrections and overshoot. Keep consistent timing distributions.

Players have been reporting getting banned for unrealistic aim, this is what detects that. I'll most likely do a deeper dive on Cerebro, because this is an extremely interesting AC mechanism and one of the more sophisticated ones that actually works well.

Summary
  • 6-layer anti-cheat: EAC + Cerebro behavioral ML + ONNX runtime + EmbarkAntiCheat + server digest validation + preloader anti-tamper
  • Cerebro is the hard one unironically - needs humanized input patterns, not just hiding what you're doing in memory
  • ONNX 1.14.1 embedded for ML inference during gameplay - includes full object detection pipeline and LSTM/attention for temporal analysis
  • Custom .mbrkcf2 protection section
  • Angelscript integration could be an attack surface
  • Reflex integration - good source for camera matrices
  • Lots of replication hooks (OnRep_*) for network sync analysis
