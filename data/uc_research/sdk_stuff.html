SDK stuff
how to auto-discover game offsets and survive patches

tired of spending 3 hours in IDA after every tuesday patch? yeah me too

this is how you build automated discovery that scans the exe for instruction patterns and extracts all your decrypt keys/offsets without manual reversing

basically: game patches -> run tool once -> everything rediscovered (GObjects, FNamePool, masks, all of it)

no more IDA hell every update

the pipeline

Code:
┌──────────────────────────────────────────────────┐
│ Layer 1: PE Headers (find valid memory regions)  │
│ - parse PE headers from target process           │
│ - get .rdata / .data / .text bounds              │
│ - filters out garbage hits later                 │
└────────────────────┬─────────────────────────────┘
                     │
┌────────────────────▼─────────────────────────────┐
│ Layer 2: AOB Scanning (find candidate locations) │
│ - IDA-style patterns like "F2 0F ?? ?? ??"      │
│ - scans whole exe for matches                    │
│ - returns list of RVAs                           │
└────────────────────┬─────────────────────────────┘
                     │
┌────────────────────▼─────────────────────────────┐
│ Layer 3: Zydis Decoder (extract actual values)   │
│ - decode instructions at each hit                │
│ - pull out immediates, RIP offsets, operands     │
│ - this is where you get the final decrypt params │
└──────────────────────────────────────────────────┘
Layer 1: PE headers

if you hardcode section boundaries you're screwed every patch

instead parse live PE headers from memory at runtime:

cpp:
// Read DOS header to find PE signature offset
uint32_t e_lfanew = 0;
mem.Read(gameBase + 0x3C, &e_lfanew, 4);

// Verify PE signature
uint32_t peSig = 0;
mem.Read(gameBase + e_lfanew, &peSig, 4);  // Must be 0x50450000 ("PE\0\0")

// Read section count and iterate section headers
uint16_t numSections = 0;
mem.Read(gameBase + e_lfanew + 6, &numSections, 2);

// Parse each section header
for (int i = 0; i < numSections; i++) {
    uint64_t secOffset = e_lfanew + 24 + optHeaderSize + (i * 40);
    uint8_t secHeader[40];
    mem.Read(gameBase + secOffset, secHeader, 40);
    
    // Extract VirtualAddress, VirtualSize, Characteristics
    uint32_t va, vsize, chars;
    memcpy(&va,    secHeader + 12, 4);
    memcpy(&vsize, secHeader + 8, 4);
    memcpy(&chars, secHeader + 36, 4);
    
    // Identify .rdata (read-only initialized data) vs .data (writable initialized data)
    if ((chars & 0x40000040) == 0x40000040 && !(chars & 0x80000000)) {
        // This is .rdata
        rdataStart = va;
        rdataEnd = va + vsize;
    }
}

now you got rdataStart, rdataEnd, dataStart, dataEnd to filter garbage hits later

Layer 2: AOB scanning

IDA-style patterns with wildcards:

Code:
F2 0F 70 05 ?? ?? ?? ?? ?? 66 0F EF 05 ?? ?? ?? ??
│  │  │  │  └ wildcard     │  │  │  │  └ wildcard
│  │  │  │    (4 bytes)     │  │  │  │    (4 bytes)
│  │  │  └ PSHUFLW RIP      │  │  │  └ PXOR RIP
└──┴──┴───┘ prefix          └──┴──┴────┘ prefix
PSHUFLW (F2 0F 70)         PXOR (66 0F EF)
matches PSHUFLW + PXOR (common in SIMD decrypt)

scanner code

cpp:
std::vector<uint64_t> SigScanner::ScanStr(const std::string& idaPattern) {
    // Parse "F2 0F 70 05 ?? ??" into bytes and mask
    std::vector<uint8_t> pattern;
    std::string mask;
    
    std::istringstream ss(idaPattern);
    std::string token;
    while (ss >> token) {
        if (token == "??" || token == "?") {
            pattern.push_back(0x00);
            mask += '?';
        } else {
            pattern.push_back(strtol(token.c_str(), nullptr, 16));
            mask += 'x';
        }
    }
    
    // Scan local module buffer
    std::vector<uint64_t> results;
    for (size_t i = 0; i < moduleBuffer.size() - pattern.size(); i++) {
        bool match = true;
        for (size_t j = 0; j < pattern.size(); j++) {
            if (mask[j] == 'x' && moduleBuffer[i + j] != pattern[j]) {
                match = false;
                break;
            }
        }
        if (match) {
            results.push_back(i);  // RVA
        }
    }
    return results;
}

important - cache the module

read entire exe into local buffer ONCE then scan that instead of spamming remote reads

cpp:
bool SigScanner::Initialize(uint64_t gameBase, uint64_t moduleSize) {
    m_moduleData.resize(moduleSize);
    
    // batch read in 4KB pages to handle missing pages gracefully
    constexpr size_t PAGE_SIZE = 0x1000;
    for (size_t offset = 0; offset < moduleSize; offset += PAGE_SIZE) {
        // if your mem lib supports batch reads use those
        // otherwise just loop with regular reads
        mem.Read(gameBase + offset, m_moduleData.data() + offset, PAGE_SIZE);
    }
}

takes 30-60 sec to read module once but scans after that are instant

Layer 3: zydis

dont hand-parse x86 instructions trust me

handrolled = pain:
  • breaks on variants
  • edge case hell
  • youll regret it

Zydis v4:
  • handles encoding automatically
  • clean operand extraction
  • works

zydis usage

cpp:
#include <Zydis.h>

struct DecodedInsn {
    ZydisDecodedInstruction raw;
    ZydisDecodedOperand ops[ZYDIS_MAX_OPERAND_COUNT];
    uint64_t rva;
    uint8_t length;
    
    // Decode immediate operand
    uint64_t imm(int n = 0) const {
        if (ops[n].type != ZYDIS_OPERAND_TYPE_IMMEDIATE) return 0;
        return ops[n].imm.value.u;
    }
    
    // Decode RIP-relative address operand
    uint64_t resolveRipRVA(int n, uint64_t gameBase) const {
        if (ops[n].type != ZYDIS_OPERAND_TYPE_MEMORY ||
            ops[n].mem.base != ZYDIS_REGISTER_RIP) return 0;
        
        ZyanU64 abs = 0;
        ZydisCalcAbsoluteAddress(&raw, &ops[n], gameBase + rva, &abs);
        return abs - gameBase;
    }
};

class FuncAnalyzer {
    std::vector<DecodedInsn> decode(const uint8_t* buf, size_t len, uint64_t rva) {
        std::vector<DecodedInsn> insns;
        
        ZydisDecoder decoder;
        ZydisDecoderInit(&decoder, ZYDIS_MACHINE_MODE_LONG_64, ZYDIS_ADDRESS_WIDTH_64);
        
        size_t offset = 0;
        while (offset < len) {
            DecodedInsn di;
            ri.rva = rva + offset;
            
            if (ZYAN_SUCCESS(ZydisDecoderDecodeBuffer(
                    &decoder, buf + offset, len - offset,
                    &di.raw, di.ops))) {
                di.length = di.raw.length;
                insns.push_back(di);
                offset += di.raw.length;
            } else {
                break;
            }
        }
        return insns;
    }
};

discovery phases

Phase 1: GObjects

sig: F2 0F 70 05 ?? ?? ?? ?? ?? 66 0F EF 05 ?? ?? ?? ??
  • find PSHUFLW + PXOR
  • decode to get shuffle immediate + encrypted RVA
  • grab XOR mask RVA and ROL amounts from nearby insns
  • store it

the decrypt basically does this:

Code:
MOV     RAX, [RIP+off1]           <- load encrypted ptr
PSHUFLW XMM0, RAX, 0x93          <- shuffle with immediate
MOVAPS  XMM1, [RIP+maskAddr]     <- load mask
PXOR    XMM0, XMM1               <- XOR decrypt
ROL64   RAX, 17                  <- rotate 17 bits
; RAX = decrypted GObjects
why it survives patches
  • devs dont change decrypt logic itself
  • only mask values and shuffle immediates change
  • instruction sequence is stable

Phase 2: FNamePool

sig: 69 D2 93 01 00 01
  • FNamePool uses FNV-1a hashing
  • constants are hardcoded in algo
  • scan for them
  • grab pool addr from nearby LEA

Phases 3-8

same pattern:
  • find insn sequence in IDA once
  • make sig with wildcards
  • scan module
  • decode to extract immediates/RVAs
  • validate
  • save

implementation notes

validation

cpp:
bool AutoConfig::ValidateGObjects(const RuntimeConfig& config) {
    // bounds check
    if (config.guobjectArrayRVA < 0x1000 || 
        config.guobjectArrayRVA >= scanner.ModuleSize()) {
        return false;
    }
    
    // section check
    if (!IsRdataRVA(config.gobjectsXorMaskRVA)) {
        return false;
    }
    
    // rotation should exist
    if (config.gobjectsRol64Amount == 0) {
        return false;
    }
    
    return true;
}

multiple candidates

cpp:
std::sort(candidates.begin(), candidates.end(), [](const auto& a, const auto& b) {
    // PSHUFB = less false positives
    bool a_shufb = (a.shufMaskRVA != 0);
    bool b_shufb = (b.shufMaskRVA != 0);
    if (a_shufb != b_shufb) return a_shufb;
    
    // higher RVA = usually code section
    return a.encryptedRVA > b.encryptedRVA;
});

live masks

some masks are in .data and get patched at runtime so read from live memory:

cpp:
void AutoConfig::ReadLiveMasks(uint64_t gameBase) {
    auto readMask = [&](const char* name, uint64_t rva, void* out, size_t size) {
        if (rva == 0) return true;
        
        // read from live memory not cache
        if (!mem.Read(gameBase + rva, out, size)) {
            std::cerr << "[Auto] Failed " << name << " at 0x" 
                      << std::hex << rva << std::dec << "\n";
            return false;
        }
        return true;
    };
    
    readMask("GObj XOR",  g_Config.gobjectsXorMaskRVA,  &g_Config.gobjectsXorMask, 16);
    readMask("GObj SHUF", g_Config.gobjectsShufMaskRVA, &g_Config.gobjectsShufMask, 16);
    // ...
}

v4 over v3 cuz:
  • newer instructions
  • better errors
  • cleaner interface

memory access libs
  • PCILeech / DMA hardware
  • KDUtils / kernel drivers
  • ReadProcessMemory / direct kernel access
  • batch read wrappers for performance

getting module base

Option A
cpp:
#include <windows.h>
HMODULE hMod = GetModuleHandleA("game.exe");  // Returns base address

Option B
  • Read PEB from target process
  • Follow InMemoryOrderModuleList
  • Extract base + module size from LDR_DATA_TABLE_ENTRY

Option C
  • supply gameBase + moduleSize manually
  • all reads go through whatever method you're using (driver/DMA/etc)

anti-cheat stuff

dont do these (gets you caught)
  • spam PE header reads
  • huge consecutive memory reads
  • GetProcAddress calls

do these instead
  • parse PE once at startup
  • batch reads in 4KB chunks (handle missing pages gracefully)
  • inline everything / dont import from DLLs
  • obfuscate function pointers
  • randomize scan timing

cpp:
int main() {
    // 1. attach to game
    uint64_t gameBase, moduleSize;
    DWORD pid;

    scanner.Shutdown();
}

stuff i learned
  • patterns from IDA might not match runtime cuz compiler optimization - always test in-game
  • Zydis > manual parsing (trust me on this one)
  • parse PE sections live dont hardcode
  • batch reads way faster than sequential
  • validate everything multiple times
  • store game version with discovered params for when stuff breaks
  • have backup patterns ready

        if (ops[n].type != ZYDIS_OPERAND_TYPE_IMMEDIATE) return 0;
        return ops[n].imm.value.u;
    }
    
    // Decode RIP-relative address operand
    uint64_t resolveRipRVA(int n, uint64_t gameBase) const {
        if (ops[n].type != ZYDIS_OPERAND_TYPE_MEMORY ||
            ops[n].mem.base != ZYDIS_REGISTER_RIP) return 0;
        
        ZyanU64 abs = 0;
        ZydisCalcAbsoluteAddress(&raw, &ops[n], gameBase + rva, &abs);
        return abs - gameBase;
    }
};

class FuncAnalyzer {
    std::vector<DecodedInsn> decode(const uint8_t* buf, size_t len, uint64_t rva) {
        std::vector<DecodedInsn> insns;
        
        ZydisDecoder decoder;
        ZydisDecoderInit(&decoder, ZYDIS_MACHINE_MODE_LONG_64, ZYDIS_ADDRESS_WIDTH_64);
        
        size_t offset = 0;
        while (offset < len) {
            DecodedInsn di;
            ri.rva = rva + offset;
            
            if (ZYAN_SUCCESS(ZydisDecoderDecodeBuffer(
                    &decoder, buf + offset, len - offset,
                    &di.raw, di.ops))) {
                di.length = di.raw.length;
                insns.push_back(di);
                offset += di.raw.length;
            } else {
                break;
            }
        }
        return insns;
    }
};

discovery phases

Phase 1: GObjects

sig: F2 0F 70 05 ?? ?? ?? ?? ?? 66 0F EF 05 ?? ?? ?? ??
  • find PSHUFLW + PXOR
  • decode to get shuffle immediate + encrypted RVA
  • grab XOR mask RVA and ROL amounts from nearby insns
  • store it

the decrypt basically does this:

Code:
MOV     RAX, [RIP+off1]           <- load encrypted ptr
PSHUFLW XMM0, RAX, 0x93          <- shuffle with immediate
MOVAPS  XMM1, [RIP+maskAddr]     <- load mask
PXOR    XMM0, XMM1               <- XOR decrypt
ROL64   RAX, 17                  <- rotate 17 bits
; RAX = decrypted GObjects
why it survives patches
  • devs dont change decrypt logic itself
  • only mask values and shuffle immediates change
  • instruction sequence is stable

Phase 2: FNamePool

sig: 69 D2 93 01 00 01
  • FNamePool uses FNV-1a hashing
  • constants are hardcoded in algo
  • scan for them
  • grab pool addr from nearby LEA

Phases 3-8

same pattern:
  • find insn sequence in IDA once
  • make sig with wildcards
  • scan module
  • decode to extract immediates/RVAs
  • validate
  • save

implementation notes

validation

cpp:
bool AutoConfig::ValidateGObjects(const RuntimeConfig& config) {
    // bounds check
    if (config.guobjectArrayRVA < 0x1000 || 
        config.guobjectArrayRVA >= scanner.ModuleSize()) {
        return false;
    }
    
    // section check
    if (!IsRdataRVA(config.gobjectsXorMaskRVA)) {
        return false;
    }
    
    // rotation should exist
    if (config.gobjectsRol64Amount == 0) {
        return false;
    }
    
    return true;
}

multiple candidates

cpp:
std::sort(candidates.begin(), candidates.end(), [](const auto& a, const auto& b) {
    // PSHUFB = less false positives
    bool a_shufb = (a.shufMaskRVA != 0);
    bool b_shufb = (b.shufMaskRVA != 0);
    if (a_shufb != b_shufb) return a_shufb;
    
    // higher RVA = usually code section
    return a.encryptedRVA > b.encryptedRVA;
});

live masks

some masks are in .data and get patched at runtime so read from live memory:

cpp:
void AutoConfig::ReadLiveMasks(uint64_t gameBase) {
    auto readMask = [&](const char* name, uint64_t rva, void* out, size_t size) {
        if (rva == 0) return true;
        
        // read from live memory not cache
        if (!mem.Read(gameBase + rva, out, size)) {
            std::cerr << "[Auto] Failed " << name << " at 0x" 
                      << std::hex << rva << std::dec << "\n";
            return false;
        }
        return true;
    };

}

  • newer instructions

  • cleaner interface

memory access libs
  • PCILeech / DMA hardware
  • KDUtils / kernel drivers
  • ReadProcessMemory / direct kernel access
  • batch read wrappers for performance

getting module base

Option A
cpp:
#include <windows.h>
HMODULE hMod = GetModuleHandleA("game.exe");  // Returns base address

Option B
  • Read PEB from target process
  • Follow InMemoryOrderModuleList
  • Extract base + module size from LDR_DATA_TABLE_ENTRY

Option C
  • supply gameBase + moduleSize manually
  • all reads go through whatever method you're using (driver/DMA/etc)

anti-cheat stuff

dont do these (gets you caught)
  • spam PE header reads
  • huge consecutive memory reads
  • GetProcAddress calls

do these instead
  • parse PE once at startup
  • batch reads in 4KB chunks (handle missing pages gracefully)
  • inline everything / dont import from DLLs
  • obfuscate function pointers
  • randomize scan timing

cpp:

stuff i learned
  • patterns from IDA might not match runtime cuz compiler optimization - always test in-game
  • Zydis > manual parsing (trust me on this one)
  • parse PE sections live dont hardcode
  • batch reads way faster than sequential
  • validate everything multiple times
  • store game version with discovered params for when stuff breaks
  • have backup patterns ready
